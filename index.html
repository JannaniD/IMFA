What is IMFA

Integrated Multi Factor Authentication System is a webportal which provides 4 levels of authentication to enable the user to be authenticated. IMFA allows the users to

  • Authenticate with 4 levels
  • Customize security solutions
  • Assure consumer identity

IMFA is the simplest, most effective way to make sure users really are who they say they are. It works by requiring users to confirm multiple factors before permitting access, instead of just an email and a password.

LEVEL 1 - Text Password

Usually text password is something that the user has to remember. The limitation in the most of the Multi Factor Authentication is that if the user forgets the text password then it is impossible to move to the next level and also if the user tries to reset the password it doesnt provide authentication.

So in this level, we have come with a type of text password that the user doesnt need to remember.

We use smart card for this level, where the card has the id number of the user. And using the given id number we will generate a random 4 digit-number, where the user will be asked to enter physically.

Here, the system will generate a random 4 digit password with the userâ€™s id number. The user has to enter the values corresponding to the positions of the random 4 digit password to authenticate themselves. This level of authentication ensures that only authorized users with the correct password can access the system.

So the first level of IMFA claims to be user friendly as the text password is generated randomly and the user no need to remember the password.

LEVEL 2 - QR Code Authentication

The QR Code Authentication is a mobile-to-web cross-login which uses qr code to authenticated with the mobile phone of the user. The QR code displayed on the web screen should be scanned in the mobile using the app and it is used to authenticate the user.

  • It is user-friendly
  • It provides higher security
  • Dictionary attacks are infeasible
  • CCP makes attacks based on hotspot analysis more challenging

    • This level of authentication involves scanning a QR code generated on the web application using a mobile device. Once scanned, the user is redirected to the login page on their mobile device,and the system authenticates the user based on their login credentials.

    This level of authentication provides an additional layer of security to prevent unauthorized access also allows the user to scan the QR code from the mobile phone. It is one of the most easier and most preferrable type of authentication.

    LEVEL 3 - Smartcard Authentication

    The smart card stores a users public key credentials and a personal identification number(PIN), which acts as the secret key to authenticate the user to the smart card. A smart card enhances security-you cant gather user details (such as a PIN) by tampering with these cards.

    There are two basic types of smart cards, contact and contactless:

    • Contact cards have a smart chip embedded in the surface of the card, and it must come into contact with the reader to work.
    • Contactless cards have a chip and antenna system that can wirelessly send the data to the reader once it comes into range.

    In this system, we use contact smartcard. Contact cards use an embedded chip to send and receive data. When the card is placed in the reader the circuit on the chip is closed and the data can be read. This allows to cards to store your information, process data, and perform tasks.

    LEVEL 4 - Biometric Authentication

    Biometric authentication is a security process that relies on the unique biological characteristics of individuals to verify they are who they say they are. Biometric authentication systems compare physical or behavioral traits to stored, confirmed, authentic data in a database.

    In this system we provide facial recognition. It is a method of biometric identification that uses that body measures, in this case, face and head, to verify the identity of a person through its facial biometric pattern and data. The technology collects a set of unique biometric data of each person associated with their face and facial expression to identify, verify and/or authenticate a person.

    Advantages of Biometric Authentication

    • Uniqueness
    • Enhanced Security
    • Accountability
    • Cost-effectiveness
